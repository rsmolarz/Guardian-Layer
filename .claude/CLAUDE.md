# Guardian-Layer ‚Äî developer context

## Stack
- pnpm monorepo. Workspaces: `artifacts/api-server`, `artifacts/guardianlayer`, `lib/db`, `lib/api-zod`, `lib/api-client-react`, `lib/integrations-*`.
- Backend: Express 5 + TypeScript, bundled to CJS via esbuild (`artifacts/api-server/build.ts`).
- Frontend: React 19 + Vite 7 + Tailwind 4 + Radix + TanStack Query + Wouter.
- DB: PostgreSQL via Drizzle ORM. Schema in `lib/db/src/schema/*.ts`, migrations in `lib/db/drizzle/*.sql`.

## Env vars that will kill the server at boot if missing
- `PORT`, `DATABASE_URL`, `NODE_ENV`, `JWT_SECRET`
- `APP_ENCRYPTION_KEY` ‚Äî rotating this after data is written causes silent decrypt failures in secure-vault, gateway, remote-machines, node-diagnostics, tailscale-imported devices. Not recoverable without re-encrypting the rows under the old key first.
- `AI_INTEGRATIONS_OPENAI_BASE_URL`, `AI_INTEGRATIONS_OPENAI_API_KEY` ‚Äî placeholder values satisfy the boot check; AI routes 500 at request time but app boots.

See `.env.example` for the full list.

## Auth + routing (post-patch)
- `authMiddleware.isPublicPath()` only gates `/api/*` paths. SPA and static assets pass through to `express.static`.
- Public `/api` endpoints are listed in `PUBLIC_PATHS` at the top of `src/middleware/auth.ts`: login, firebase, webauthn/login/*, medinvest/*, did/*, health, healthz, /metrics.
- IP guard auto-allows 127.0.0.1 + RFC1918 ranges. External IPs get rate-tracked; abusive ones get blocked with escalating durations.
- Global rate limiter: 1000 req / 15 min per IP.

## Encryption (post-patch)
Six files use AES-256-GCM with a key derived from `sha256(APP_ENCRYPTION_KEY)`: `lib/seed-remote-machines.ts`, `lib/seed-tailscale.ts`, `routes/gateway.ts`, `routes/node-diagnostics.ts`, `routes/remote-maintenance.ts`, `routes/secure-vault.ts`. Each throws at both module load (IIFE) and per-call if `APP_ENCRYPTION_KEY` is missing. No fallbacks.

## Schema
The Drizzle schema in `lib/db/src/schema/*.ts` may drift ahead of the committed migrations under `lib/db/drizzle/`. If you see `column "..." does not exist` at runtime, run:
```
pnpm --filter @workspace/db install
pnpm --filter @workspace/db run push
```
`push` is interactive ‚Äî say YES to additive changes (ADD column, CREATE table, ADD constraint), STOP and ask a human before any DROP, TRUNCATE, or RENAME.

## Superadmin
Hardcoded in `src/lib/seed-users.ts`: email `rsmolarz@rsmolarz.com`, username `rsmolarz`, role `superadmin`. Password comes from `SUPERADMIN_INITIAL_PASSWORD` env var; the seed re-syncs the DB hash to this value on every restart if they differ. To change the password for real, remove the env var (not just the value) before the UI change, or the next restart reverts your change.

## Build + deploy
- Dev: `pnpm dev` (Replit auto-runs via the artifact.toml config).
- Build (both frontend and backend): `pnpm run build`.
- The backend bundle externalizes some native deps (ssh2, pg-native, etc.) ‚Äî `pnpm install` is required on any deploy target to fetch them, even though the CJS bundle is pre-built.

## Conventions
- Don't commit `.env`, `dist/`, `node_modules/`, `backups/`.
- Don't edit `dist/` files directly ‚Äî regenerated by build. All changes go in source.
- Catalog versions: shared deps live in `pnpm-workspace.yaml` under `catalog:`. Bump there.
- `activityLoggerMiddleware` is mounted on `/api` ‚Äî DB write happens on every request. Keep in mind for hot paths.
