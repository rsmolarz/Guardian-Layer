## PHASE 1 — CRITICAL BUG FIXES (do these first)

### Fix 1: Cloud File Storage
Migrate all file uploads from the local filesystem to Cloudinary. Install the 'cloudinary' npm package. Create a /server/utils/storage.js utility module. Update every upload endpoint (profile photos, lesson images, course thumbnails, document uploads) to upload to Cloudinary and store the returned secure_url in the database. Remove all local /uploads folder dependencies. Add CLOUDINARY_CLOUD_NAME, CLOUDINARY_API_KEY, and CLOUDINARY_API_SECRET to the .env file with placeholder values.

### Fix 2: PDF and DOCX Upload Parsing
Fix document upload and parsing. Configure 'multer' with a 10MB limit accepting application/pdf and application/vnd.openxmlformats-officedocument.wordprocessingml.document. Use 'pdf-parse' for PDFs and 'mammoth' for DOCX files. Add proper error handling returning a 400 with a descriptive message if parsing fails. Log all parsing errors server-side with the filename and error message.

### Fix 3: Quiz Score Persistence
Fix quiz score saving. Ensure the POST /api/quiz-submissions endpoint writes to the database with: user_id, quiz_id, score, answers (JSON), attempt_number, and submitted_at. Add a unique constraint on (user_id, quiz_id, attempt_number). On the quiz results page, always fetch from GET /api/quiz-submissions/:quizId/my-result — never from local state only. Return the saved record in the POST response.

### Fix 4: Video Embedding
Fix YouTube and Vimeo embedding in the lesson editor. Write a parseVideoUrl(url) utility that detects YouTube (youtube.com/watch?v=, youtu.be/) and Vimeo (vimeo.com/) links and converts them to proper embed URLs (https://www.youtube.com/embed/VIDEO_ID with rel=0&modestbranding=1). Sanitize all URLs to prevent XSS. Render as a responsive 16:9 iframe wrapper (padding-bottom: 56.25%) in the lesson viewer.

### Fix 5: User Registration and Email Verification
Fix user registration: (1) hash all passwords with bcrypt at 10 rounds, (2) generate a UUID verification token on registration, (3) store token with a 24-hour expiry in a email_verification_tokens table, (4) send a verification email using Nodemailer with a link to /verify?token=TOKEN, (5) clicking the link marks the user's email_verified field as true. Block unverified users from enrolling in courses with a friendly prompt to verify. Add a "Resend verification email" button on the learner dashboard.

### Fix 6: Password Reset Flow
Implement a complete password reset flow: (1) POST /api/auth/forgot-password accepts an email, generates a secure token via crypto.randomBytes(32).toString('hex'), stores it with a 1-hour expiry in a password_reset_tokens table, and sends a reset email, (2) GET /reset-password?token=TOKEN shows a form, (3) POST /api/auth/reset-password validates the token is unexpired, accepts a new password (min 8 chars), hashes it, updates the user, deletes the token, and redirects to login with a success message.

### Fix 7: Role-Based Access Control
Implement RBAC. Add a role column to the users table with values: learner, instructor, admin (default: learner). Create a requireRole(...roles) middleware that checks req.user.role and returns 403 if unauthorized. Apply requireRole('admin') to all /admin/* routes. Apply requireRole('instructor','admin') to course creation and editing routes. Add a Users Management page in the admin panel to view all users and change their roles. Seed one default admin user using credentials from .env (ADMIN_EMAIL, ADMIN_PASSWORD).

### Fix 8: Course Progress Calculation
Fix progress tracking. Create a lesson_completions table (user_id, lesson_id, completed_at, PRIMARY KEY(user_id, lesson_id)). When a learner clicks "Mark Complete" on a lesson, upsert a record. Calculate progress as: (COUNT of completed lessons for this user in this course) / (total published lessons in course) * 100. Display the correct % on the learner dashboard and inside each course. Trigger certificate generation automatically when a course reaches 100% and the learner has a passing quiz score (>= course passing_score setting, default 70%).
